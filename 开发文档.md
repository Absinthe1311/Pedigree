# Pedigree 开发步骤文档

> 技术栈：React + TypeScript（前端）· NestJS（后端）· PostgreSQL（数据库）· D3.js（可视化）
> 环境：WSL2 Ubuntu 22.04

---

## 第一阶段：环境准备

### 1.1 安装 NVM 和 Node.js

NVM 是 Node.js 的版本管理工具，通过它来安装 Node.js。

```bash
# 安装 NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载终端配置（必须执行）
source ~/.bashrc

# 验证 NVM 安装成功
nvm --version

# 安装 Node.js 18 LTS
nvm install 18
nvm use 18
nvm alias default 18

# 验证
node --version    # 应输出 v18.x.x
npm --version     # 应输出 9.x.x 或 10.x.x
```

### 1.2 安装 Git

```bash
sudo apt install git -y

# 配置个人信息（替换为你自己的）
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

# 验证
git --version
```

### 1.3 安装 PostgreSQL

```bash
# 安装
sudo apt install postgresql postgresql-contrib -y

# 启动服务
sudo service postgresql start

# 验证服务状态
sudo service postgresql status
# 看到 "online" 即为正常运行
```

### 1.4 初始化数据库

```bash
# 进入 PostgreSQL 命令行
sudo -u postgres psql
```

在 psql 提示符下执行以下命令：

```sql
-- 创建项目专用用户
CREATE USER pedigree WITH PASSWORD 'pedigree123';

-- 创建数据库并指定所有者
CREATE DATABASE pedigree_db OWNER pedigree;

-- 验证创建成功
\l

-- 退出
\q
```

验证连接：

```bash
psql -h localhost -U pedigree -d pedigree_db
# 输入密码 pedigree123
# 看到 pedigree_db=# 提示符即成功
# 输入 \q 退出
```

### 1.5 安装 NestJS CLI

```bash
npm install -g @nestjs/cli

# 验证
nest --version    # 应输出 10.x.x
```

## 第三阶段：数据库构建

### 3.1 创建建表文件

```bash
cat > ~/Pedigree/database/schema.sql << 'EOF'
-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id          SERIAL PRIMARY KEY,
    username    VARCHAR(50)  UNIQUE NOT NULL,
    email       VARCHAR(100) UNIQUE NOT NULL,
    password    VARCHAR(255) NOT NULL,
    created_at  TIMESTAMP DEFAULT NOW()
);

-- 族谱表
CREATE TABLE IF NOT EXISTS families (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    surname     VARCHAR(50)  NOT NULL,
    revised_at  DATE,
    owner_id    INTEGER REFERENCES users(id),
    created_at  TIMESTAMP DEFAULT NOW()
);

-- 成员表
CREATE TABLE IF NOT EXISTS members (
    id          SERIAL PRIMARY KEY,
    family_id   INTEGER REFERENCES families(id) ON DELETE CASCADE,
    name        VARCHAR(100) NOT NULL,
    gender      CHAR(1) CHECK (gender IN ('M','F')),
    birth_year  INTEGER,
    death_year  INTEGER,
    bio         TEXT,
    created_at  TIMESTAMP DEFAULT NOW()
);

-- 血缘关系表
CREATE TABLE IF NOT EXISTS blood_relations (
    id          SERIAL PRIMARY KEY,
    parent_id   INTEGER REFERENCES members(id) ON DELETE CASCADE,
    child_id    INTEGER REFERENCES members(id) ON DELETE CASCADE,
    relation    VARCHAR(10) CHECK (relation IN ('父子','父女','母子','母女')),
    UNIQUE (parent_id, child_id)
);

-- 婚姻关系表
CREATE TABLE IF NOT EXISTS marriage_relations (
    id          SERIAL PRIMARY KEY,
    member_a_id INTEGER REFERENCES members(id) ON DELETE CASCADE,
    member_b_id INTEGER REFERENCES members(id) ON DELETE CASCADE,
    UNIQUE (member_a_id, member_b_id)
);

-- 邀请协作表
CREATE TABLE IF NOT EXISTS invitations (
    id          SERIAL PRIMARY KEY,
    family_id   INTEGER REFERENCES families(id) ON DELETE CASCADE,
    inviter_id  INTEGER REFERENCES users(id),
    invitee_id  INTEGER REFERENCES users(id),
    status      VARCHAR(10) DEFAULT 'pending'
                CHECK (status IN ('pending','accepted','rejected')),
    created_at  TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_members_name   ON members(name);
CREATE INDEX IF NOT EXISTS idx_blood_parent   ON blood_relations(parent_id);
CREATE INDEX IF NOT EXISTS idx_blood_child    ON blood_relations(child_id);
CREATE INDEX IF NOT EXISTS idx_members_family ON members(family_id);
EOF
```

### 3.2 执行建表

```bash
psql -h localhost -U pedigree -d pedigree_db -f ~/Pedigree/database/schema.sql
# 输入密码 pedigree123
```

### 3.3 验证建表成功

```bash

```

```sql
-- users
-- families
-- members
-- blood_relations
-- marriage_relations
-- invitations

\q
```

## 第四阶段：后端配置

### 4.1 创建环境变量文件

```bash
cat > ~/Pedigree/backend/.env << 'EOF'
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=pedigree
DB_PASSWORD=pedigree123
DB_DATABASE=pedigree_db
JWT_SECRET=pedigree_secret_key_change_in_production
JWT_EXPIRES_IN=7d
EOF
```

### 4.2 配置数据库连接

编辑 `backend/src/app.module.ts`，添加 TypeORM 配置：

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { TypeOrmModule } from '@nestjs/typeorm';

@Module({
  imports: [
    ConfigModule.forRoot({ isGlobal: true }),
    TypeOrmModule.forRoot({
      type: 'postgres',
      host: process.env.DB_HOST,
      port: parseInt(process.env.DB_PORT),
      username: process.env.DB_USERNAME,
      password: process.env.DB_PASSWORD,
      database: process.env.DB_DATABASE,
      entities: [__dirname + '/**/*.entity{.ts,.js}'],
      synchronize: false,
    }),
  ],
})
export class AppModule {}
```

### 4.3 生成各功能模块

```bash
cd ~/Pedigree/backend

nest g module auth
nest g module users
nest g module families
nest g module members
nest g module relations
nest g module query
nest g module invitations
```

## 第五阶段：前端配置

### 5.1 配置 API 请求封装

创建 `frontend/src/api/client.ts`：

```typescript
import axios from 'axios';

const client = axios.create({
  baseURL: 'http://localhost:3000',
  timeout: 10000,
});

// 自动注入 JWT Token
client.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 统一处理 401 未授权
client.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default client;
```

### 5.2 配置路由结构

编辑 `frontend/src/main.tsx`：

## 第六阶段：日常开发流程

### 每次开发前启动服务

每次开发需要开启两个终端窗口：

```bash
# 终端 1：启动后端
cd ~/Pedigree/backend
sudo service postgresql start    # 确保数据库在运行
npm run start:dev                # 后端监听 http://localhost:3000

# 终端 2：启动前端
cd ~/Pedigree/frontend
npm run dev                      # 前端监听 http://localhost:5173
```

### 开发顺序建议

按以下顺序推进，每完成一个模块用 Postman 测试接口后再进入下一个：

```
后端开发顺序：
1. auth 模块      → 注册、登录、JWT 签发与验证
2. users 模块     → 用户信息管理
3. families 模块  → 族谱增删改查、权限控制
4. members 模块   → 成员增删改查、模糊搜索
5. relations 模块 → 血缘关系、婚姻关系管理
6. query 模块     → 祖先查询、后代查询、亲缘路径查询
7. invitations 模块 → 邀请协作功能

前端开发顺序：
1. LoginPage      → 登录注册表单
2. DashboardPage  → 族谱列表、统计数据
3. FamilyListPage → 族谱管理、邀请功能
4. MemberPage     → 成员列表、模糊搜索、增删改查
5. TreeChart 组件 → D3.js 树状图
6. QueryPage      → 祖先查询、亲缘路径查询
```
